Übung 10: TypeScript

1. Static classes in TypeScript

in TypeScript gibt es statische Klassen, die ähnlich wie in Java und C# verwendet werden können. Eine statische Klasse ist eine Klasse, deren Methoden und Eigenschaften direkt auf die Klasse selbst und nicht auf eine Instanz der Klasse zugreifen. Im Vergleich zu Java und C# bietet TypeScript jedoch einige spezifische Merkmale und Unterschiede. TypeScript ist eine Erweiterung von JavaScript und bietet eine optionale Typprüfung, während Java und C# eigene, statisch typisierte Sprachen sind. TypeScript ermöglicht es Entwicklern, JavaScript-Code mit Typprüfungen und anderen Funktionen moderner Sprachen zu schreiben, aber auch direkt JavaScript-Code auszuführen. Hier ist ein Beispiel, das die Typprüfung in TypeScript demonstriert: 


        let userName: string = "John Doe";
        userName = 123; // Compilerfehler: Type '123' is not assignable to type 'string'.

TypeScript ist auch flexibler bei der Verwendung von dynamischen Typisierungen im Vergleich zu Java und C#. Es unterstützt auch einige moderne Sprachfunktionen wie Async/Await und Decorators. Zusammenfassend bietet TypeScript eine engere Integration mit JavaScript und ermöglicht es Entwicklern, JavaScript-Code mit Typprüfungen und anderen Funktionen moderner Sprachen zu schreiben, während Java und C# eigene, statisch typisierte Sprachen sind.

2. Fehler finden mit TypeScript

das fehlerbehaftete JavaScript-Programm. enthält mehrere Fehler:
+ Unklare Typen: Der Typ von x und y ist nicht definiert und wird daher automatisch als any getypt.
+ Inkonsistente Typen: Inder Funktion add können unterschiedliche Typen übergeben werden, was zu einer Inkonsistenz bei der Verarbeitung führen kann.
+ Überflüssiger Code: Die Verwendung von var ist überflüssig und kann Verwirrung stiften.
Hier ist ein ergänztes TypeScript-Programm: 


        const add = (x: number, y: number): number => x + y;
        const equals = (x: any, y: any): boolean => x === y;
        
        console.log(add(1, 2));
        console.log(add(true, true));
        console.log(add(true, false));
        
        let x: number = 1;
        let y: number = 1;
        console.log(add(equals(x, y), equals(y, x)));

typeScript meldet folgende Fehler:
+ In der Funktion add werden Typen übergeben, die nicht number sind, was zu einer Typfehlermeldung führt.
+In der Funktion equals werden Typen übergeben, die nicht explizit getypt sind, was zu einer Meldung über unklare Typen führt.
